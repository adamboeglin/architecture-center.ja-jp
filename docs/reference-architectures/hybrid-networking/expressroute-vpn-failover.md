### <a name="vnet-and-gatewaysubnet"></a><span data-ttu-id="c37a1-143">VNet と GatewaySubnet</span><span class="sxs-lookup"><span data-stu-id="c37a1-143">VNet and GatewaySubnet</span></span>

<span data-ttu-id="c37a1-144">ExpressRoute 仮想ネットワーク ゲートウェイと VPN 仮想ネットワーク ゲートウェイは、同じ VNet 内に作成します。</span><span class="sxs-lookup"><span data-stu-id="c37a1-144">Create the ExpressRoute virtual network gateway and the VPN virtual network gateway in the same VNet.</span></span> <span data-ttu-id="c37a1-145">つまり両者は、*GatewaySubnet* という同じサブネットを共有する必要があります。</span><span class="sxs-lookup"><span data-stu-id="c37a1-145">This means that they should share the same subnet named *GatewaySubnet*.</span></span>

<span data-ttu-id="c37a1-146">*GatewaySubnet* という名前のサブネットが既に VNet に含まれている場合は、/27 以上のアドレス空間があることを確認してください。</span><span class="sxs-lookup"><span data-stu-id="c37a1-146">If the VNet already includes a subnet named *GatewaySubnet*, ensure that it has a /27 or larger address space.</span></span> <span data-ttu-id="c37a1-147">既存のサブネットが小さすぎる場合は、次の PowerShell コマンドを使用して、そのサブネットを削除します。</span><span class="sxs-lookup"><span data-stu-id="c37a1-147">If the existing subnet is too small, use the following PowerShell command to remove the subnet:</span></span>

```

<span data-ttu-id="c37a1-148">**GatewaySubnet** という名前のサブネットが VNet に含まれていない場合は、次の PowerShell コマンドを使用して新しいサブネットを作成します。</span><span class="sxs-lookup"><span data-stu-id="c37a1-148">If the VNet does not contain a subnet named **GatewaySubnet**, create a new one using the following PowerShell command:</span></span>

$vnet = Set-AzureRmVirtualNetwork -VirtualNetwork $vnet
```
